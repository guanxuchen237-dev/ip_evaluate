## 🔬 二、平台归一化效果

### 2.1 归一化前后对比

**问题**: 起点与纵横平台用户基数差异巨大

| 平台 | 平均月票(原始) | 平均收藏(原始) | 平均推荐(原始) |
|------|---------------|---------------|---------------|
| 起点 | 43,196 | 1,624,453 | 3,848,038 |
| 纵横 | 914 | 25,040,085 | 850,111 |

**解决方案**: 计算缩放因子，将纵横数据对齐到起点量级

```
纵横缩放因子 = 起点平均月票 / 纵横平均月票 = 43,196 / 914 = 47.22x
```

### 2.2 归一化效果

| 指标 | 归一化前 | 归一化后 |
|------|---------|---------|
| 月票范围 | 7 ~ 1,044,969 | 331 ~ 3,034,269 |
| 数据分布 | 双峰分离 | 相对统一 |
| 平台可比性 | 差 | 好 |

**结论**: 平台归一化成功消除了起点/纵横的量级差异，使模型能够公平学习两个平台的特征。

**平台归一化核心代码**:
```python
class PlatformNormalizer:
    def fit(self, df):
        qidian_avg = df[df['platform']=='Qidian']['monthly_tickets'].mean()
        zongheng_avg = df[df['platform']=='Zongheng']['monthly_tickets'].mean()
        self.zongheng_scale_factor = qidian_avg / (zongheng_avg + 1)
        
    def transform(self, df):
        mask = df['platform'] == 'Zongheng'
        df.loc[mask, 'monthly_tickets_normalized'] = (
            df.loc[mask, 'monthly_tickets'] * self.zongheng_scale_factor
        )
        return df
```
