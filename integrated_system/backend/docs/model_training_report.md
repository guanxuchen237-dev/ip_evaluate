# 月票预测模型训练报告

## 一、模型训练概述

本项目训练了两个版本的XGBoost月票预测模型，分别基于不同类型的数据源，以对比验证历史数据对预测性能的影响。

### 1.1 训练目标
构建高精度的月票预测模型，能够基于书籍的历史表现、当前排名、时间特征等预测下个月的月票数量。

### 1.2 数据源对比

| 属性 | 实时监测模型 | 历史数据模型 |
|------|-------------|-------------|
| **数据表** | `novel_realtime_tracking`<br>`zongheng_realtime_tracking` | `novel_monthly_stats`<br>`zongheng_book_ranks` |
| **时间跨度** | 短期实时数据 | 2020-2025年 (5年) |
| **样本量** | 267条 | 11,234条 |
| **书籍数量** | 105本 | 1,904本 |
| **数据采集** | 爬虫实时追踪 | 月度统计归档 |

## 二、特征工程

两个模型共提取 **21个特征**，分为五大类别：

### 2.1 基础特征 (4个)

| 特征名 | 类型 | 说明 | 数据来源 |
|--------|------|------|----------|
| `log_word_count` | 数值 | 字数对数变换，处理长尾分布 | word_count |
| `word_growth` | 数值 | 字数增长量，反映更新活跃度 | word_count差分 |
| `rank_num` | 数值 | 当前排名，越小表示越热门 | rank_on_list |
| `platform_code` | 类别 | 平台编码：起点=1，纵横=0 | platform |

### 2.2 时间特征 (4个)

| 特征名 | 类型 | 说明 |
|--------|------|------|
| `quarter` | 类别 | 季度 (1-4)，捕捉季节性趋势 |
| `is_year_end` | 二元 | 是否年末 (11-12月)，年末通常月票竞争激烈 |
| `is_year_start` | 二元 | 是否年初 (1-2月)，新年重置效应 |
| `months_since_start` | 数值 | 书籍发布月数，生命周期特征 |

### 2.3 历史统计特征 (6个)

| 特征名 | 类型 | 计算方法 | 意义 |
|--------|------|----------|------|
| `hist_mean` | 数值 | 历史月票均值 | 长期平均水平 |
| `hist_max` | 数值 | 历史最高月票 | 峰值潜力 |
| `hist_min` | 数值 | 历史最低月票 | 底线保障 |
| `hist_std` | 数值 | 历史标准差 | 波动性指标 |
| `last_3_mean` | 数值 | 近3个月平均 | 短期趋势 |
| `last_3_max` | 数值 | 近3个月最高 | 近期峰值 |

### 2.4 近期趋势特征 (4个)

| 特征名 | 类型 | 计算方法 | 意义 |
|--------|------|----------|------|
| `last_month_tickets` | 数值 | 上月月票数 | 基准参考值 |
| `last_month_rank` | 数值 | 上月排名 | 排名趋势 |
| `rank_change` | 数值 | 排名变化量 | 排名上升/下降趋势 |
| `mom_growth` | 数值 | (本月-上月)/上月 | 环比增长率，动量指标 |

### 2.5 类别与生命周期 (3个)

| 特征名 | 类型 | 说明 |
|--------|------|------|
| `category_code` | 类别 | 题材编码：玄幻、都市、仙侠等 |
| `is_new_book` | 二元 | 新书标志 (≤3个月)，新书通常有流量扶持 |
| `is_mature` | 二元 | 成熟书标志 (≥12个月)，进入稳定期 |

## 三、模型训练过程

### 3.1 实时监测模型训练

```python
# 训练脚本: train_ticket_model.py
# 数据提取逻辑
- 从实时追踪表获取最近7天数据窗口
- 计算窗口内的均值、标准差、最大值、最小值
- 提取时间特征（星期几、小时、是否周末）
- 目标：预测未来7天平均月票
```

**训练参数：**
```python
{
    'objective': 'reg:squarederror',
    'max_depth': 8,
    'learning_rate': 0.05,
    'n_estimators': 200,
    'subsample': 0.8,
    'colsample_bytree': 0.8
}
```

### 3.2 历史数据模型训练

```python
# 训练脚本: train_historical_model.py
# 数据提取逻辑
- 从月度统计表获取2020-2025年数据
- 按书籍分组计算时序特征
- 使用expanding窗口计算历史统计量
- 使用rolling窗口计算近期趋势
- 目标：预测下个月月票（对数变换处理）
```

**训练参数：**
```python
{
    'objective': 'reg:squarederror',
    'max_depth': 10,
    'learning_rate': 0.03,
    'n_estimators': 500,
    'subsample': 0.85,
    'colsample_bytree': 0.85,
    'reg_alpha': 0.1,
    'reg_lambda': 1.0
}
```

**关键优化：**
- 使用对数变换处理月票长尾分布 (`y_log = log1p(y)`)
- 特征标准化 (StandardScaler)
- 早停机制 (early_stopping_rounds=30)

## 五、结论与建议

### 5.1 核心结论

🏆 **历史数据模型完胜**

1. **数据量优势**：42倍训练数据 (11,234 vs 267)
2. **特征丰富度**：21个精心设计的时序特征
3. **时间跨度**：5年历史数据捕捉长期趋势
4. **泛化能力**：在新数据上MAPE仅6.56%

### 5.2 实时监测模型失败原因

1. **数据量不足**：267条样本难以学习稳健模式
2. **时间窗口短**：无法捕捉长期趋势和季节性
3. **特征单一**：过度依赖当前值，缺乏历史上下文
4. **过拟合风险**：小数据集容易过拟合特定时间段

### 5.3 生产环境建议

- **采用历史数据模型** 作为默认预测引擎
- **保留实时数据** 用于模型在线更新和监控
- **定期重训练**：每月使用新数据更新模型
- **A/B测试**：对关键预测进行多模型对比验证
